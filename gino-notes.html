Lesson 1: The Basics of the Web and HTML

The web and how it works
The web consists of a large number of computers spread all over the world that communicate with one another. Information requested to the server through a web browser via the internet. These requests use what's called Hypertext Transfer Protocol (HTTP) Request. The server(s) replies to the request by returning the HTML document file which the web browser displays (interprets) using the (HTTP) Response. Watch this video to have a better understanding of this concept.
What is a server and what is it used for?
Server is a special computer that is used to manage access to a centralized resource. One of its functions (or services if you will) is to accept requests from another computer on the internet
Computers Don't Think Like Humans
Computers just interpret programmed instructions programmed by programmers literally. This makes them very unforgiving so if a programmer is not skillful he can cause huge problems in a program.
HTML Tags and Elements

HTML documents are made of HTML elements. When writing HTML, we tell browsers the type of each element by using HTML tags

Inline and Block Elements
Inline tags are tags that do not create an invisible box around the content. There is no space created. Examples of inline elements: img, span, span, b and em tags Block elements form an "invisible box" around the content inside of them creating extra spaces. By default, browsers always place a line break before and after the < div > element. However, this can be changed in the Cascading Style Sheet.

Lesson 2: Creating a Structured Document with HTML

The "tree-like structure" of HTML
The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them. The web browser parses the HTML source code and creates an object tree based on it. In other words, the Document Object Model is the browser's internal representation of HTML source code.
Special Text Editors (for programming)
When writing code, programmers use special text editors (like Sublime Text for example). These editors are designed with features that basically enable programmers to avoid typos and increase their productivity. For example, some text editors will automatically generate a closing HTML tag when you write an opening tag.
Relationship between indentend HTML and boxes
When you read an HTML document as text, you noticed indentations throughout the document. The more indented an element is, the more deeply nested its corresponding "box" is.
